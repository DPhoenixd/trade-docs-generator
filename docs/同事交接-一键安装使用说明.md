# P.I/P.L一点腾使用说明

写给接手外贸单据的同事。

先说人话版结论：这个工具不是让你研究电脑的，是让你少改 Excel 单元格的。你只要会上传截图、检查数字、点生成，就能把一份 P.I 先做出来。

它主要干三件事：

- 识别客户截图里的颜色、公司色号、数量、单位、价格
- 根据面料编号查面料数据库，自动换算 KG、Yard、USD/KG、USD/Yard
- 用模板生成 P.I，后续 P.L 用码单细码单独处理

也就是说，它不是魔法棒。它更像一个很勤快但偶尔眼神不太好的实习生：它能帮你抄、算、填，但最后你一定要看一眼。

## 第一次打开

你会拿到一个压缩包：

`PIPL-YiDianTeng-portable.zip`

操作很简单：

1. 先解压
2. 打开解压后的文件夹
3. 双击 `PIPL-YiDianTeng.exe`
4. 等几秒，浏览器会自己打开工具页面

不用打开命令行。

不用安装 Python。

不用安装 Node.js。

如果电脑弹出安全提醒，说这个程序来自互联网、要不要运行，点“仍要运行”。这是 Windows 对所有陌生 exe 的常规疑心病，不是工具爆炸了。

## 每次做订单的正确姿势

页面分三步，不要跳着来。

### 第 1 步：上传客户截图

这里上传的是客户发来的需求截图，不是已经做好的 P.I。

截图里一般会有这些东西：

- STYLE
- COLOR NAME
- COLOR CODE
- FABRIC QTY
- Fabric price
- Total amount

工具会尽量识别出：

- 颜色名，比如 `CAMEL`
- 公司色号，比如 `8#`
- 数量，比如 `45`
- 单位，比如 `Yard` 或 `Meter`
- 单价，比如 `$4.57`

识别完以后，先看一眼表格。

重点看这几个：

- 色号有没有变形：公司色号应该长得像 `8#`、`21+3#`
- 数量有没有少 0 或多 0
- 单位是不是对的：客户给的是 Y 还是 m
- 一行颜色有没有被识别成两行

如果识别错了，直接在表格里改。这个地方就是给你改的，不要怕。

### 第 2 步：补订单信息

这一页是人工信息区。机器不可能知道客户这次 PO 是什么，所以你必须填。

重点填这些：

- `PO NO.`
- Buyer
- Buyer’s address
- Terms of payment
- Delivery Time
- Port of Destination
- 面料编号

面料编号那里，输入比如 `6529`，点检索，选对应的面料。

选中面料以后，工具才知道这块布的：

- 成分
- 克重
- 幅宽
- 量化
- 纸筒空差

然后它才能把客户给的 Yard / Meter 换算成 KG，也才能算出 USD/KG 和 USD/Yard。

这里有个重点：`USD/KG` 是内部审核价格，`USD/Yard` 是客户常看的码价。两个都要看，别只看金额总数。

### 第 3 步：审查并生成

到了这一步，不要急着点生成。

先看右边实时预览，重点盯这几个：

- PO 对不对
- Buyer 对不对
- 面料编号对不对
- 颜色和色号对不对
- KG 数量对不对
- Yard 数量对不对
- USD/KG 是否合理
- USD/Yard 是否合理
- Total Amount 是否合理
- Deposit 30% 是否合理
- Balance 是否合理

都没问题，再点 `生成 P.I`。

现在 P.I 和 P.L 是分开的。

P.I 用客户需求截图来做。

P.L 需要码单细码，也就是 LOT、ROLL、Net Weight 那些东西。没有码单细码，就不要强行生成 P.L。强行生成出来也不靠谱。

## 模板到底是干嘛的

模板不是业务数据。

模板不是业务数据。

模板不是业务数据。

重要的事情说三遍，因为 Excel 最容易在这里把人带沟里。

P.I 模板只负责版式，比如表头、收款信息、格式、字体、位置。它不是让工具照抄旧订单。

所以正确逻辑是：

- 客户截图和你手动修改的内容优先
- 面料数据库负责换算和面料信息
- P.I 模板只负责输出成像样的 Excel

如果你先填好了订单，再上传模板，正常情况下不应该把你填的东西覆盖掉。如果发生了，先别继续生成，截图告诉维护的人。

## 面料数据库是什么

面料数据库就是这工具的“布料脑子”。

里面存了每个面料编号对应的：

- 中文/英文名称
- 成分
- 幅宽
- 克重
- 量化
- 纸筒 + 空差
- 价格规则

如果你输入面料编号查不到，不是你不会用，大概率是数据库里没有这款布。

这种情况不要硬做，找维护的人更新数据库。

## 常见翻车现场

### 1. 双击 exe 没反应

先等 10 秒。

这个工具第一次启动会在后台开一个本地服务，电脑慢一点时不是秒开。

如果还是没打开，再双击一次。

还不行的话，打开浏览器，手动输入：

`http://127.0.0.1:8787/`

### 2. 页面打开了，但上传截图没有识别出东西

先看页面有没有红色错误提示。

如果没有错误，但识别结果是空的，通常是这几种情况：

- 截图太糊
- 表格线太淡
- 截图被裁掉了表头
- 图片里字太小
- 客户截图格式太离谱

解决方法：

- 尽量上传完整表格截图
- 截图里要包含表头
- 不要只截半张
- 必要时把识别结果手动新增进去

### 3. 颜色识别出来了，但色号怪怪的

公司色号应该是这种：

- `8#`
- `13#`
- `21+3#`

如果你看到 `#8#`、`COLOR: 8`、`8` 这种奇怪格式，直接改成 `8#`。

### 4. 金额是 0

一般是因为没有价格，或者没有选面料编号。

检查：

- 截图里有没有单价
- 单价有没有识别出来
- 面料编号有没有选中
- 单位是不是 Yard / Meter / KG

### 5. USD/KG 看起来很怪

先别生成。

USD/KG 是内部审核价，错了可能不是小问题。

重点检查：

- 单位是不是识别错了
- 量化是不是选错面料导致的
- 客户给的是 USD/Yard 还是 USD/Meter
- 数量是不是少了一个 0

### 6. 上传 P.I 模板以后内容变了

正常情况下不应该。

模板只提供格式，不应该抢业务数据。

如果真的发生，先点清空重新来，不要拿错误数据继续生成。然后把截图发给维护的人。

## 什么东西必须人工确认

别完全相信工具。

下面这些必须人工看：

- PO NO.
- Buyer
- Buyer’s address
- Terms of payment
- Delivery Time
- Port of Destination
- 面料编号
- USD/KG
- USD/Yard
- Total Amount
- Deposit 30%
- Balance

尤其是地址、付款方式、目的港。错了不是 Excel 难看，是后面付款、清关、发货都会变麻烦。

## 这个工具适合做什么

适合：

- 根据客户截图快速整理 P.I
- 自动换算 KG / Yard
- 快速核对 KG 单价和码价
- 减少手改 Excel 的重复劳动
- 让订单数据先有一个干净版本

不适合：

- 盲生成
- 不看预览直接发客户
- 拿旧模板当新订单数据
- 在面料库没有数据时硬算

## 最后一句

这个工具的目标不是让你变成电脑高手。

它的目标是：别再为了一个 P.I，在 Excel 里改到怀疑人生。

但它再聪明，也只是工具。客户、价格、地址、付款方式这些关键内容，最后还是要你这个真人点头。
